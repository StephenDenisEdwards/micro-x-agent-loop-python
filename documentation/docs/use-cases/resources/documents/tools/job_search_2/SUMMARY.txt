================================================================================
JOB SEARCH CONSOLE APP - PROJECT SUMMARY
================================================================================

PROJECT CREATED: tools/job_search_2/

PURPOSE:
  Python console application that replicates the Claude agent's job search 
  workflow using direct API access to minimize LLM costs.

KEY OBJECTIVE:
  Search Gmail for JobServe emails + LinkedIn for contract roles, score each 
  job against criteria (1-10 scale), and generate markdown reports - all with 
  ZERO LLM API calls.

HOW IT WORKS
================================================================================

WORKFLOW:
  1. Load config + search criteria from documents folder
  2. Search Gmail for JobServe emails (last 24 hours)
  3. Parse emails with regex to extract: title, company, location, rate, IR35
  4. Search LinkedIn for contract roles (multiple keyword passes)
  5. Score ALL jobs against 5 factors:
     - Technology match (30% weight)
     - Seniority level (25% weight)
     - Day rate (20% weight)
     - Sector preference (15% weight)
     - Location match (10% weight)
     - Bonuses: AI/ML, Healthcare, Outside IR35, Architecture
  6. Filter jobs (only include 5+/10 scores)
  7. Generate markdown report:
     - Top 10 ranked list
     - JobServe job details section
     - LinkedIn job details section
     - Summary statistics + observations + recommendations

OUTPUT:
  File: documents/todays-jobs-YYYY-MM-DD.md (e.g., todays-jobs-2026-03-01.md)

EXECUTION TIME:
  <5 seconds total for 100 jobs (vs. 30-60 seconds for Claude agent)

COST:
  $0 per run (vs. $3-5 per run for Claude agent)

================================================================================
SCORING LOGIC
================================================================================

WEIGHTS:
  Technology:   30%  (# of core tech matches)
  Seniority:    25%  (Senior/Lead/Architect keywords)
  Rate:         20%  (£/day or salary equivalent)
  Sector:       15%  (Healthcare, Finance, Legal, etc.)
  Location:     10%  (London or Remote UK)
  Bonuses:     +1.5  (AI/ML, Healthcare, Outside IR35, Architecture)

SCORE RANGES:
  9-10:  Perfect match  → Apply immediately
  7-8:   Strong match   → High priority
  5-6:   Good match     → Worth considering
  <5:    Weak match     → Excluded from report

TECHNOLOGY MATCHING:
  Tech keywords tracked: .NET, C#, Azure, Python, Microservices, Blazor, 
                         AI/ML, Kubernetes, Docker, React, Angular, etc.
  
  Score by count:
    0 matches → 2/10
    1 match   → 4/10
    2 matches → 6/10
    3 matches → 8/10
    4+ matches → 10/10

SENIORITY DETECTION:
  Regex-based keyword matching:
    "Senior" | "Lead" | "Principal" | "Architect" → 10/10
    "Junior" | "Graduate" | "Entry Level" → 2/10
    Unknown / Mid-level → 6/10

RATE SCORING (£/day):
  £600+  → 10/10
  £500+  → 8/10
  £400+  → 6/10
  £300+  → 4/10
  <£300  → 2/10
  Unknown → 4/10

SECTOR MATCHING:
  Preferred: Healthcare, Finance, Legal, Industrial, Energy
  1 sector match    → 7/10
  2+ sector matches → 10/10
  Unknown sector    → 5/10 (neutral)

LOCATION MATCHING:
  London or Remote UK    → 10/10
  Hybrid (London/UK)     → 9/10
  Remote (any)           → 6/10
  Unknown                → 4/10

BONUSES:
  AI/ML keywords           +0.5
  Healthcare domain        +0.5
  Outside IR35             +0.5
  Architecture/Design lead +0.3
  Maximum bonus cap        +1.5

================================================================================
MCP INTEGRATION
================================================================================

WHAT IS MCP?
  Message Passing Protocol - stdio-based communication with AI services.
  Allows direct API access without web browsers or heavy SDKs.

ARCHITECTURE:
  mcp_client.py provides:
    - MCPClient (base class)
    - GmailMCPClient (gmail_search, gmail_read methods)
    - LinkedInMCPClient (linkedin_jobs, linkedin_job_detail methods)

DRY RUN MODE (Current):
  - gmail_client.py and linkedin_client.py are placeholders
  - Return empty job lists
  - Useful for testing without MCP servers

PRODUCTION MODE:
  Update main.py imports:
    from gmail_client_mcp import GmailClientMCP
    from linkedin_client_mcp import LinkedInClientMCP
  
  Start MCP servers (separate terminals):
    # Terminal 1
    mcp-gmail --stdio
    
    # Terminal 2
    mcp-linkedin --stdio
    
  Run app:
    python main.py

REQUEST/RESPONSE FORMAT:
  Request (to MCP server):
    {"method": "gmail_search", "params": {"query": "from:jobserve after:2026-02-28"}}
  
  Response (from MCP server):
    {"result": [{"id": "msg123", "from": "...", "subject": "...", "body": "..."}]}

================================================================================
EMAIL PARSING
================================================================================

JOBSERVE EMAIL STRUCTURE:
  Subject: "[Job Title] - [Company] - [Location]"
  Body:    Full job specification with rate, duration, requirements

EXTRACTION PATTERNS:
  Job Title:  From subject line (first part before hyphens)
  Company:    From subject line (second part)
  Location:   From subject line (third part)
  
  Rate:       £([\d,\-]+)(?:\s*/\s*day|/d)?
              Examples: £500/day, £500-650, £500,000
  
  Duration:   (\d+\s*(?:months?|weeks?|days?))
              Examples: 6 months, 12 weeks, 3 days
  
  Reference:  (?:Ref|Reference)[:=]?\s*([A-Z0-9]+)
              Examples: Ref: ABC123, Reference:XY789, JX12345
  
  IR35:       Case-insensitive substring matching
              "Outside IR35" → Outside IR35
              "Inside IR35" → Inside IR35
              (not found) → Not specified

================================================================================
REPORT FORMAT
================================================================================

FILE NAMING:
  Strict format: todays-jobs-YYYY-MM-DD.md
  Examples: todays-jobs-2026-03-01.md
  
  NOT allowed:
    todays-jobs-2026-03-01-v1.md (version suffix)
    todays-jobs-2026-03-01-01-30.md (timestamp)

STRUCTURE:
  1. Header
  2. Top 10 Best Matches (2-line entries with scores)
  3. JobServe Jobs Section (full specs)
  4. LinkedIn Jobs Section (full specs)
  5. Summary Statistics (counts, distributions, observations)

TOP 10 FORMAT:
  1. **[Job Title](#anchor)** - Score: 8/10
     £500-650/day, Remote UK, 6 months. .NET/Azure, microservices.

DETAILED JOB FORMAT (JobServe):
  ### Job Title
  **Score: X/10**
  
  **Location:** [location]
  **Rate:** £XXX/day
  **Duration:** X months
  **IR35:** Inside/Outside/Not specified
  **Posted:** JobServe, Ref: ABC123
  
  **Summary:**
  [One paragraph: role description, key tech, responsibilities]
  
  **Links:**
  - [Job Spec](url)
  - [Apply](url)
  
  **Why this score:**
  [One paragraph: what matches, what doesn't, constraints]

DETAILED JOB FORMAT (LinkedIn):
  ### Company - Job Title
  **Score: X/10**
  
  **Company:** [name]
  **Location:** [location]
  **Type:** Contract - Remote
  **Sector:** [sector]
  **Posted:** [time ago]
  
  **Summary:**
  [One paragraph]
  
  **Link:** [View Job](url)
  
  **Why this score:**
  [One paragraph with bold constraints like "**Permanent role, not contract**"]

SUMMARY STATISTICS SECTION:
  - Total jobs found (JobServe + LinkedIn breakdown)
  - Jobs scoring 5+/10
  - Average score
  - Top 8 technologies with counts
  - All sectors mentioned
  - Contract vs. Permanent split
  - Location distribution
  - IR35 status breakdown
  - 5-7 key observations about market trends
  - 5-6 prioritized recommendations
  - Timestamp of report generation

STAGED FILE WRITING:
  write_file()  → Header + Top 10
  append_file() → JobServe section
  append_file() → LinkedIn section
  append_file() → Summary statistics
  
  (Avoids token limit issues with large files)
